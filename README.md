## Краткая информация

Задача выполнена с использованием фреймворка Laravel 12, языка PHP 8.3, базы данный Sqlite.

Эндпойнты работают как /api/tasks (с префиксом api) - маршруты прописаны в routes/api.php

Так и просто /tasks, без префикса api - прописаны в routes/web.php. Для того, чтобы работали без префикса api, пришлоось отключить проверку csrf, что небезопасно.

<h1> Установка </h1>
<ol>
    <li>Клонируем репозиторий - git clone git@github.com:vvk2001github/to-do-list.git</li>
    <li>Устанавливаем необходимые пакеты - composer install</li>
    <li>Выполняем миграции - php artisan migrate</li>
    <li>Запускаем тестовый сервер - php artisan serve</li>
</ol>

<h1> Использование </h1>

<h2>1. Создание задачи.</h2>

Для создания задачи необходимо методом POST вызвать эндпойнт /tasks, передав теле запроса следующие параметры:
<ul>
    <li><b>title</b> - название задачи, строка, обязательное поле, максимальная длина - 255 символов;</li>
    <li><b>descrption</b> - описание задачи, строка, необязательное поле;</li>
    <li><b>status</b> - статус задачи, целое число, необязательное поле, допустимые значения от 0 до 2.</li>
</ul>

<b>Пример тела запроса.</b>
```json
{
    "title": "Таск01",
    "description": "Задача один",
    "status": 2
}
```
При успешном выполнении запроса статус ответа будет 201 Created, в противном случае статус ответа 422 Unprocessable Entity.

<b>Пример ответа.</b>
```json
{
    "data": {
        "id": 28,
        "title": "Таск01",
        "description": "Задача один",
        "status": 2
    }
}
```
Описание полей объекта data в ответе.
<ul>
    <li><b>id</b> - идентификатор задачи, целое число;</li>
    <li><b>title</b> - название задачи, строка;</li>
    <li><b>descrption</b> - описание задачи, строка;</li>
    <li><b>status</b> - статус задачи, целое число.</li>
</ul>


<h2>2. Получение списка задач.</h2>
Приполучении списка задач реализована пагинация (постраничное отображение).

Для получения списка задач необходимо методом GET вызвать эндпойнт /tasks, передав теле запроса следующие параметры:
<ul>
    <li><b>per_page</b> - количество задач на странице, получаемых за один раз, целое число, необязательное поле, по умолчанию - 15;</li>
    <li><b>page</b> - номер желаемой страницы, целое число, необязательное поле.</li>
</ul>

<b>Пример тела запроса.</b>
```json
{
    "per_page": 2,
    "page": 2
}
```
При успешном выполнении запроса статус ответа будет 200 Ok, в противном случае статус ответа 422 Unprocessable Entity.

<b>Пример ответа.</b>
```json
{
    "data": [
        {
            "id": 11,
            "title": "test01",
            "description": null,
            "status": 0
        },
        {
            "id": 12,
            "title": "test123567",
            "description": null,
            "status": 2
        }
    ],
    "pagination": {
        "current_page": 2,
        "last_page": 10,
        "per_page": 2,
        "total": 20
    }
}
```
Описание полей объектов в массиве data соответсвует описанию полей в ответе присоздании задачи.

Описание полей объекта pagination.
<ul>
    <li><b>current_page</b> - номер текущей страницы, целое число;</li>
    <li><b>last_page</b> - номер последней страницы, целое число.</li>
    <li><b>per_page</b> - количество задач на странице, получаемых за один раз, целое число;</li>
    <li><b>total</b> - общее количество страниц, получаемых за один раз, целое число.</li>
</ul>
Нумерация страниц начинается с единицы.


<h2>3. Получение одной задачи.</h2>
Для получения одной задачи необходимо методом GET вызвать эндпойнт /tasks/{taskId}, где {taskId} - идентификатор желаемой задачи. Тело запроса пустое.

При успешном выполнении запроса статус ответа будет 200 Ok. Если задачи с таким идентификатором не существует, то статус ответа будет 404 Not Found. В остальных случаях - статус ответа 422 Unprocessable Entity.

<h2>4. Изменениее одной задачи.</h2>
Для изменения одной задачи необходимо методом PUT вызвать эндпойнт /tasks/{taskId}, где {taskId} - идентификатор желаемой задачи.
В теле запроса можно передать следующие <b>необязательные</b> поля:
<ul>
    <li><b>title</b> - название задачи, строка, максимальная длина - 255 символов;</li>
    <li><b>descrption</b> - описание задачи, строка;</li>
    <li><b>status</b> - статус задачи, целое число,  допустимые значения от 0 до 2.</li>
</ul>

<b>Пример тела запроса.</b>
```json
{
    "title": "Таск01",
    "description": "Задача один",
    "status": 2
}
```
При успешном выполнении запроса статус ответа будет 200 Ok. Если задачи с таким идентификатором не существует, то статус ответа будет 404 Not Found. В остальных случаях - статус ответа 422 Unprocessable Entity.

Формат ответа аналогичен ответу при создании задачи.


<h2>5. Удаление одной задачи.</h2>
Для удаления одной задачи необходимо методом DELETE вызвать эндпойнт /tasks/{taskId}, где {taskId} - идентификатор желаемой задачи. 

При успешном выполнении запроса, статус ответа будет 204 No Content и пустое тело ответа. 
Если задачи с таким идентификатором не существует, то статус ответа будет 404 Not Found. 
В остальных случаях - статус ответа 422 Unprocessable Entity.
